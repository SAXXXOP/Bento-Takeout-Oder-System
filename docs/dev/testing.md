# テスト手順（本番を壊さない）

## 目的
改修時に「本番データを壊さず」安全に確認する。

---

## 推奨：テスト用コピーで検証
1. 本番スプレッドシートをコピー（テスト環境）
2. テスト用フォームを紐づけ
3. Script Properties はテスト用に設定（トークンはダミーでもOK）
4. トリガーをテスト用にだけ設定

---

## 最低テストケース
- 新規予約（通常）
- 電話番号未入力（要確認になる）
- 締切後送信（要確認/受付不可の挙動）
- 変更予約（予約番号あり）
- LINE（任意）：送信/受信の疎通

---

## ログを使った確認
- 注文一覧：行が正しく追加/更新される
- 要確認一覧：対象が出る/消える
- 予約札/当日まとめ：指定日で生成できる
- 実行履歴：例外なし

---

## 注意
- 本番にトリガーを二重設定しない（多重実行の原因）
- テスト後はトリガーを削除
