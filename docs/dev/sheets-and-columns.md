# シート/列仕様（CONFIG.SHEET / CONFIG.COLUMN）

## 目的
テンプレ前提（シート名/列位置）を明文化し、運用・改修で壊さない。

---

## 必須シート（例）
- 注文一覧
- 当日まとめ
- 予約札
- 顧客名簿
- メニューマスタ
- ★要確認一覧
- ログ
- 設定

---

## 注文一覧（例：列位置固定）
このプロジェクトは `CONFIG.COLUMN` で列位置を固定している前提です。  
実際の列名はリポジトリの CONFIG を参照してこのページに追記してください。

最低限、運用で意識する列（例）：
- TIMESTAMP（受付時刻）
- ORDER_NO（予約No）
- NAME（氏名）
- TEL（電話）
- PICKUP_DATE（受け取り：表示用 "M/D... / 時刻"）
- PICKUP_DATE_RAW（受け取り日：内部Date）
- DETAILS（注文明細）
- STATUS（空/無効/★要確認）
- REASON（要確認理由など）
- NOTE（備考）

---

## メニューマスタ
- 商品名、略称、価格、（自動返信用表示）など
- 変更すると集計/表示が変わるため、更新時はテスト推奨

---

## 変更時の注意
- 列の挿入/削除は危険（位置固定前提を壊す）
- どうしても変える場合は CONFIG とコードを同時変更し、テストを必ず行う
