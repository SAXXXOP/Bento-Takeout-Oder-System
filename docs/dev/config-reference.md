# CONFIG 参照（設定値の意味）

## 目的
設定（CONFIG）とフォーム/シートの前提を一覧化し、変更時の事故を防ぐ。

---

## CONFIG.SHEET
- 各シート名を定義
- ここを変えると参照先が変わる

## CONFIG.COLUMN
- 注文一覧などの列位置を定義（A=TIMESTAMP, B=ORDER_NO... のような固定）

## CONFIG.FORM
- フォーム質問タイトル（質問文）を定義
- フォーム側の文言を変えたらここも合わせる

---

## 運用メモ
- フォーム編集（質問文の変更）は最も壊れやすい
- シート列の変更は次に壊れやすい

→ 変更前に `dev/testing.md` のテスト手順で安全確認
