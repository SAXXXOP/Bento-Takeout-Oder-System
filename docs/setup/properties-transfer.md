# Script Properties の値移植（JSONエクスポート/インポート）

## 目的
テンプレ → 店舗 へ **「値だけ」安全に移植**する（キー漏れ/打ち間違いを減らす）。

---

## 推奨フロー
1. テンプレ（または既存店舗）で **エクスポート**（JSON）
2. 店舗ごとに値を差し替え
3. 店舗側で **インポート**
4. 動作確認（フォーム送信テスト）

---

## JSONフォーマット（例）
```json
{
  "LINE_TOKEN": "xxxx",
  "WEBHOOK_KEY": "yyyy",
  "BACKUP_FOLDER_ID": "zzzz",
  "LOG_LEVEL": "INFO",
  "LOG_MAX_ROWS": "5000"
}
```

---

## 実行手順（メニューがある場合）
- スプレッドシートの「導入ツール」等に次がある想定：
  - Script Properties をエクスポート
  - Script Properties をインポート

※ 実装のメニュー名/関数名に合わせて、READMEやこのページに追記してください。

---

## 実装が無い場合（最小手動）
- エクスポート：Apps Script の「プロジェクトの設定」画面でキー/値をコピー
- インポート：店舗側で貼り付け（数が多いと事故が増えるので非推奨）

---

## 事故を防ぐコツ
- テンプレはダミー値を入れておく（空でも可）
- 店舗側の JSON を Git ではなく **安全な共有手段**で管理
- 運用者が複数いる場合は「誰が変更したか」メモを残す
