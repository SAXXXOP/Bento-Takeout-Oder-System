# 運用開始前チェックリスト

## 目的
初日トラブルを防ぐため、導入時に「最低限ここまで」を揃える。

---

## A. 事前準備（テンプレ → 店舗）
- [ ] 店舗用スプレッドシートをコピーした
- [ ] フォームがこのスプレッドシートに紐づいている
- [ ] [ ] 必須シートが揃っている（注文一覧/当日まとめ/予約札/顧客名簿/メニューマスタ/★要確認一覧/ログ）

## B. 権限（承認）
- [ ] Apps Script 初回承認を完了した
- [ ] Drive（バックアップ）や UrlFetch（LINE）など使う機能分の承認が通っている

## C. Script Properties
- [ ] 使う機能の必須値を設定した
  - [ ] LINE：`LINE_TOKEN`, `WEBHOOK_KEY`
  - [ ] バックアップ：`BACKUP_FOLDER_ID`
- [ ] `LOG_LEVEL`, `LOG_MAX_ROWS` を設定（推奨）

## D. トリガー
- [ ] `onFormSubmit` をフォーム送信時に設定した
- [ ] 日次準備（任意）：`dailyPrepTrigger` を設定した（または手動運用方針を決めた）
- [ ] バックアップ（任意）：`backupSpreadsheetDaily` を設定した

## E. 動作確認
- [ ] フォームからテスト送信→注文一覧に記録される
- [ ] 要確認一覧が必要に応じて更新される
- [ ] 予約札/当日まとめが手動で作成できる
- [ ] ログに記録が残る

## F. 運用ルール
- [ ] ステータス（有効/無効/★要確認）の判断基準を決めた
- [ ] ノーショウ対応文面/連絡先を決めた（自動返信にも反映する場合は改修）

---

完了したら：`operations/daily-check.md` の流れで初日運用へ。
